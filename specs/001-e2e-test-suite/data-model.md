# Data Model: End-to-End Test Suite

**Date**: 2026-04-26 | **Feature**: 001-e2e-test-suite

This document describes the entities, state, and cleanup model for the E2E test suite. No new production entities are introduced ‚Äî the test suite reads and writes to existing Firestore collections in the `rio-test` named database.

---

## Firestore Collections (in `rio-test` database)

All collection names are identical to production ‚Äî isolation is by database name only.

### `{theme}_topics` (e.g., `planet_protectors_topics`)

| Field | Type | Notes |
|-------|------|-------|
| `topics_id` | string | Document ID; UUID |
| `topics` | array of strings | List of topic titles generated by WF1 |
| `created_at` | timestamp | Set by WF1 on creation |

**Test assertions**: Doc exists after WF1 run; `topics` is non-empty; no title duplicates across two successive runs.

---

### `{theme}_stories` (e.g., `planet_protectors_stories`)

| Field | Type | Notes |
|-------|------|-------|
| `topics_id` | string | Document ID (same as `topics_id` from WF1) |
| `title` | string | Story title |
| `description` | string | Story body |
| `moral` | string | Story moral/lesson |
| `image_url` | string | GCS URL; populated after WF3 |
| `audio_url` | string | GCS URL; populated after WF4 |
| `created_at` | timestamp | Set by WF2 on creation |

**Test assertions**: All fields present after full pipeline; `image_url` and `audio_url` start with `gs://kutty_bucket/test/` in E2E test context.

---

### `activities_v1`

| Field | Type | Notes |
|-------|------|-------|
| document ID | string | Auto-generated |
| `story_id` | string | References story doc ID (`topics_id`) |
| `activity_type` | string | `mcq` \| `art` \| `science` \| `moral` |
| `content` | map | Activity-type-specific JSON content |
| `activity_image_url` | string | GCS URL for the activity's generated image; set by WF5 for art/science/moral; may be null for MCQ |
| `geval_score` | float | DeepEval GEval score (0.0‚Äì1.0); >= 0.7 to pass |
| `created_at` | timestamp | Set by WF5 |

**Test assertions**: 4 activities exist per story_id after WF5; each `geval_score >= 0.7`; `activity_type` values are exactly `{mcq, art, science, moral}`; `activity_image_url` is present and starts with `gs://kutty_bucket/test/` for art, science, and moral activities.

---

### `workflow_checkpoints`

| Field | Type | Notes |
|-------|------|-------|
| document ID | string | LangGraph thread_id |
| checkpoint data | map | LangGraph internal state (opaque) |

**Test assertions (retry/interrupt tests)**: Checkpoint exists while workflow is interrupted; checkpoint is deleted after successful resume + completion.

## Test Artifacts (GCS + local)

### GCS: `gs://kutty_bucket/test/images/{story_id}.png`

Generated by WF3 during E2E tests. Must not collide with production paths (`images/{story_id}.png`).

**Lifecycle**: Created during `test_wf3_*` or full pipeline tests. Deleted by `cleanup_storage` fixture after each test function.

---

### GCS: `gs://kutty_bucket/test/audio/{story_id}.mp3`

Generated by WF4 during E2E tests. Must not collide with production paths (`audio/{story_id}.mp3`).

**Lifecycle**: Created during `test_wf4_*` or full pipeline tests. Deleted by `cleanup_storage` fixture after each test function.

---

### Local: `test/reports/run_{YYYYMMDD_HHMMSS}.json`

JSON report file written at the end of each pytest session. Not cleaned up ‚Äî kept for CI artifact upload.

**Schema**: See `research.md ¬ß5`.

## Cleanup Strategy

| Resource | When | How |
|----------|------|-----|
| Firestore story docs | After each test | Delete by `topics_id` from `{theme}_stories` |
| Firestore topic docs | After each test | Delete by `topics_id` from `{theme}_topics` |
| Firestore activity docs | After each test | Query `activities_v1` where `story_id in created_story_ids` |
| Firestore checkpoints | After each test | Delete by `thread_id` from `workflow_checkpoints` |
| GCS test blobs | After each test | Delete all blobs prefixed `test/images/` and `test/audio/` matching test run |
| JSON reports | Never (kept) | Uploaded as CI artifacts |

**Teardown order**: Storage blobs ‚Üí Firestore activities ‚Üí Firestore checkpoints ‚Üí Firestore stories ‚Üí Firestore topics. Order matters to avoid orphaned references.
